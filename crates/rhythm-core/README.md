# rhythm-core

`rhythm-core` は、固定小数点演算でリズム生成と外部同期を行うためのコアライブラリです。
実装は「計算基盤」「自律更新」「同期ロジック」「通信連携」を分離し、
挙動の変更点をテストで追跡しやすい構造を重視しています。

## 何を提供するか

- 固定小数点ベースの位相・テンポ計算
- 自律リズム生成 (`update`)
- BPM ヒント付き同期 (`sync`)
- BPM ヒントなしパルス同期 (`sync_pulse`)
- ループバック・マルチキャストを使った送受信ユーティリティ (`comm`)

## 設計方針

- 浮動小数点ではなく、`u16` / `i16` を中心とした固定小数点を利用する
- 「内部状態更新」と「外部入力同期」を明確に分ける
- 位相はラップ算術を前提に扱い、境界ケースを通常経路として実装する
- 同期の収束性は、正常系だけでなくロス・遅延・ラップ境界を含めて検証する

## 主要モジュール

- `fixed_math.rs`
	- `BpmQ8`, `PhaseU16`, `SinQ15` などの型を定義
	- テンポ制約、補間、位相差、LUT 参照など計算プリミティブを集約
- `lib.rs`
	- `RhythmGenerator` と `RhythmMessage` を公開
	- `update` / `sync` / `sync_pulse` の状態遷移と同期ロジックを実装
- `comm.rs`
	- UDP ループバック・マルチキャスト送受信
	- 送信側/受信側のステップ API (`UdpMulticastSyncSender` / `UdpMulticastSyncListener`)

## 同期ロジックの考え方

### BPM ヒントあり同期 (`sync`)

- 遅延補償: 受信時刻との差分を使って相手位相を補正
- 2点観測: 観測点の時刻差だけでなく、`beat_count` 差分も使って BPM を推定
- 追従: ロック中は観測 BPM とヒント BPM を段階的にブレンドして更新
- 位相補正: 範囲外テンポは倍音/分周へ正規化し、必要に応じて四半周期候補へ吸着

### BPM ヒントなし同期 (`sync_pulse`)

- パルス間隔列からテンポを推定し、指数平均で平滑化
- 一定時間ヒントが欠落した場合はロックを解除して再キャリブレーション
- 位相補正はヒントあり同期と同様に遅延補償を経由して適用

## 通信連携 (`comm`)

- 送信 API は、内部更新を進めつつ「位相グリッドを跨いだ瞬間」にのみ発信
	- 単位分割では拍境界（`phase=0`）
	- 複数分割では分割位相グリッド
- 受信 API は、1ステップで `update + recv + sync` をまとめて実行
- example はこの API を薄く呼び出すだけにし、ロジックの重複を避ける

## テスト戦略

テストは次の区分で保守します。

- 値域確認: ラップ算術、拍カウント、固定小数点変換の基本保証
- 正常系: 低頻度入力や通常同期での収束確認
- 異常系: 範囲外テンポ、ヒント欠落、分周処理
- 実用系: ロス・遅延ジッタ注入下での同期維持
- 明示検証: `beat_count` ラップ境界（`u32` 周回）での BPM 推定整合

## メンテナンス指針

- 同期ロジックを変更したら、必ず次をセットで更新する
	- 正常系テスト
	- ロス/ジッタ系テスト
	- ラップ境界の明示検証テスト
- example 側にロジックを複製せず、`comm` のステップ API に寄せる
- ドキュメントは「実装の意図」と「不変条件」を優先し、
	調整されやすい定数値の説明は最小限に留める
